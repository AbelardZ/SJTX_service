# 公网暴露接口（Ubuntu）

## 1. 安装依赖

```bash
sudo apt update
sudo apt install -y python3 python3-venv nginx certbot python3-certbot-nginx
cd /opt
sudo mkdir -p stock-rag && sudo chown -R $USER:$USER stock-rag
```

将项目同步到 `/opt/stock-rag` 后执行：

```bash
cd /opt/stock-rag
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install fastapi uvicorn requests numpy pypdf sentence-transformers pymupdf rapidocr-onnxruntime
```

## 2. 启动服务（先本地验证）

```bash
export API_KEY="替换成你的强口令"
source .venv/bin/activate
uvicorn api_server:app --host 0.0.0.0 --port 8000
```

健康检查：

```bash
curl http://127.0.0.1:8000/healthz
```

## 3. 使用 systemd 托管

创建 `/etc/systemd/system/stock-rag.service`：

```ini
[Unit]
Description=Stock RAG API Service
After=network.target

[Service]
User=ubuntu
WorkingDirectory=/opt/stock-rag
Environment=API_KEY=替换成你的强口令
ExecStart=/opt/stock-rag/.venv/bin/uvicorn api_server:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable stock-rag
sudo systemctl restart stock-rag
sudo systemctl status stock-rag
```

## 4. Nginx 反向代理 + HTTPS

创建 `/etc/nginx/sites-available/stock-rag`：

```nginx
server {
    listen 80;
    server_name 你的域名;

    client_max_body_size 50m;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用站点：

```bash
sudo ln -sf /etc/nginx/sites-available/stock-rag /etc/nginx/sites-enabled/stock-rag
sudo nginx -t
sudo systemctl reload nginx
```

申请证书：

```bash
sudo certbot --nginx -d 你的域名
```

## 5. 接口调用示例

### 5.1 数据抓取入库
```bash
curl -X POST "https://你的域名/crawl" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: 你的API_KEY" \
  -d '{"stock_code":"600536","stock_name":"中国软件","days":365}'
```

### 5.2 构建 RAG 向量数据库
将抓取的公告及研报内容切块并向量化，存入本地向量数据库：
```bash
curl -X POST "https://你的域名/rag/build" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: 你的API_KEY" \
  -d '{
    "stock_code":"600536",
    "stock_name":"中国软件",
    "model": "BAAI/bge-small-zh-v1.5",
    "chunk_size": 700,
    "overlap": 120
  }'
```
*注：此接口由于涉及 PDF 解析和 Embedding 推理，耗时可能较长，建议可以在生产环境 Nginx 中调高 `proxy_read_timeout`。*

### 5.3 查询 RAG 数据库 (公网接口核心)
基于用户提问，在向量数据库中检索最相似的文档块：
```bash
curl -X POST "https://你的域名/rag/query" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: 你的API_KEY" \
  -d '{
    "stock_code": "600536",
    "stock_name": "中国软件",
    "q": "公司近期在人工智能领域有什么布局？",
    "top_k": 5
  }'
```

### 5.4 全链路流水线（Pipeline）
一次性执行抓取并构建向量数据库：
```bash
curl -X POST "https://你的域名/pipeline" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: 你的API_KEY" \
  -d '{
    "stock_code":"600536",
    "stock_name":"中国软件",
    "days": 365
  }'
```

## 6. Nginx 长连接与超时配置建议

因为 RAG 数据库构建（`/rag/build` 和 `/pipeline`）需要消耗较长时间进行 OCR 与向量计算，建议在 Nginx 中补充超时设置：

```nginx
server {
    listen 80;
    server_name 你的域名;

    client_max_body_size 50m;
    
    # 防止由于后台处理时间过长导致 504 Gateway Time-out
    proxy_read_timeout 1800s;
    proxy_connect_timeout 1800s;
    proxy_send_timeout 1800s;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        # ... 其它配置 ...
    }
}
```
